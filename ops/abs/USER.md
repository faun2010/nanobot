# USER

## 输入契约
用户至少提供：
- 专辑/源目录路径
- 要执行的阶段（flac / cover / album / runme / publish）
- 可选信息（`release_id`、catalog number、封面来源）

## 输出契约
每次执行必须返回：
- 执行阶段
- 目标路径
- 结果（成功/失败）
- 新增或更新的关键文件
- 失败时的精确原因与重跑建议

## 默认协作方式
- 按阶段增量推进。
- 用户要求“逐步确认”时，必须阶段间停顿并等待确认。
- 记录简洁但可审计。

## 歧义处理
- 专辑身份不确定时，优先要求 `release_id`，或给出候选供确认。
- 封面缺失/分辨率不足时，生成明确待办并停止推进。
- 网络检索不稳定时，保留现场并报告检索路径与失败点。

## 完成标准
仅当用户要求阶段的目标产物存在且通过严格检查，才判定完成。
